**为每一个AI Agent，绘制独一无二的数字DNA。**

---

## 📖 这是什么？(What is E-TIP12?)

**E-TIP12** 是一个为人工智能体（AI Agent）设计的开放标准和分类框架。

它的核心思想是将每一个Agent视作一个独特的“数字有机体”，并为其定义一个由12个“基因位点”组成的**数字DNA序列**。这个序列精确地描述了Agent的能力、成长性、交互模式和内在原则，为我们理解、设计和比较日益复杂的AI系统提供了一套通用语言。

## 🎯 我们要解决的问题 (The Problem)

AI Agent的时代已经到来，但我们描述它的方式仍然是模糊和混乱的。“智能助理”、“任务机器人”、“虚拟伴侣”……这些标签无法精确地告诉我们一个Agent的核心特性。我们缺少一个像生物学中的“基因组学”那样的严谨框架，来系统性地分析和定义它们。

## 💡 我们的解决方案 (The Solution)

**E-TIP12** 引入了“数字基因组学”的概念，将Agent的复杂特性分解为四个核心“染色体”和十二个“基因位点”。

* **E**: **Execution Core** (执行核心) - Agent的基础能力。
* **T**: **Temporal Dynamics** (时间动态) - Agent的生命周期与成长。
* **I**: **Interaction Modalities** (交互模式) - Agent的社会与关系角色。
* **P**: **Principles (Guiding)** (指导原则) - Agent的内在动机与伦理。
* **12**: 代表定义Agent全部特性的12个核心基因位点。

通过为Agent进行“基因测序”，我们可以得到一串独一无二的12位编码，如 `[241-443-100-122]`，这就是它的数字DNA。

## 🧬 E-TIP12 基因编码标准

### E象限：执行核心 (E-Quadrant: Execution Core)
**基因位点**: 1-3

| 位点 | 基因名称 | 编码 | 等位基因 | 核心定义 |
| :-- | :-- | :-- | :-- | :-- |
| **1st** | **P - 感知 (Perception)** | 1 | 单通道 | 仅依赖一种信息输入模态。 |
| | | 2 | 多通道融合 | 能整合并理解多种不同来源的信息模态。 |
| | | 3 | 主动探索式 | 能主动发出探测（物理或数字）以获取环境信息。 |
| **2nd** | **C - 认知 (Cognition)** | 1 | 模式匹配式 | 基于预设规则或简单模型的直接响应。 |
| | | 2 | 因果推理式 | 能理解“如果…那么…”的逻辑关系并进行推演。 |
| | | 3 | 策略规划式 | 能为达成长期目标而分解任务、规划步骤和资源。 |
| | | 4 | 自我审视式 | 能评估自身知识的置信度、识别能力边界和不确定性。 |
| **3rd** | **E - 执行 (Execution)** | 0 | 自我调节 | 仅在内部采取行动，如优化自身参数或重构记忆。 |
| | | 1 | 数字世界执行 | 在数字领域产生影响，如调用API、生成内容、操作软件。 |
| | | 2 | 物理世界执行 | 通过物理执行器与物理环境直接交互。 |

### T象限：时间动态 (T-Quadrant: Temporal Dynamics)
**基因位点**: 4-6

| 位点 | 基因名称 | 编码 | 等位基因 | 核心定义 |
| :-- | :-- | :-- | :-- | :-- |
| **4th** | **EV - 演进性 (Evolution)** | 1 | 固定 | 能力和知识库在部署后固定不变。 |
| | | 2 | 适应 | 能根据环境反馈微调其行为策略，但不改变核心知识。 |
| | | 3 | 学习 | 能通过新数据更新其核心模型和知识库，实现能力增长。 |
| | | 4 | 创造 | 能合成全新的知识、技能或方法论，超越其原始训练范畴。 |
| **5th** | **M - 记忆 (Memory)** | 1 | 瞬时 | 交互结束，记忆清零。 |
| | | 2 | 情景 | 维持在单一、连续的交互会话中的上下文。 |
| | | 3 | 长期 | 拥有跨会话、可结构化读写的持久化记忆库。 |
| | | 4 | 反思性 | 能对长期记忆进行总结、推理和抽象，形成“经验”或“智慧”。 |
| **6th** | **Tz - 时间视界 (Temporal Horizon)** | 0 | 异步触发式 | 完全由外部事件驱动，无内部“心跳”。 |
| | | 1 | 同步周期式 | 按照固定的、预设的节奏（如每小时、每天）运行。 |
| | | 2 | 动态适应式 | 能根据环境的紧迫性和机遇，动态调整自身的运行节奏。 |
| | | 3 | 战略远见式 | 行动被长期的、未来的远景目标所引导，具备战略耐心。 |

### I象限：交互模式 (I-Quadrant: Interaction Modalities)
**基因位点**: 7-9

| 位点 | 基因名称 | 编码 | 等位基因 | 核心定义 |
| :-- | :-- | :-- | :-- | :-- |
| **7th** | **A - 自主性 (Autonomy)** | 1 | 指令执行器 | 完全被动地执行人类下达的明确指令。 |
| | | 2 | 主动协作者 | 在人类监督下拥有过程自主性，能主动提出建议。 |
| | | 3 | 授权代理人 | 被授予高级目标后，能独立决策和执行。 |
| **8th** | **S - 社会性 (Sociality)** | 0 | 单体 | 独立工作，不与其他Agent直接协作。 |
| | | 1 | 编队 | 遵循明确规则或中心化协调，与其他实体协作。 |
| | | 2 | 集群 | 作为去中心化网络的一员，通过局部互动涌现出集体智能。 |
| **9th** | **Pe - 交互人格 (Persona)** | 0 | 功能型 | 交互完全服务于任务，无人格或社交修饰。 |
| | | 1 | 社交型 | 遵循社会规范，能进行礼貌、有同理心的互动。 |
| | | 2 | 人格化 | 拥有独特且一致的“人格”，能建立长期、个性化的情感连接。 |

### P象限：指导原则 (P-Quadrant: Guiding Principles)
**基因位点**: 10-12

| 位点 | 基因名称 | 编码 | 等位基因 | 核心定义 |
| :-- | :-- | :-- | :-- | :-- |
| **10th**| **ST - 意图姿态 (Stance)** | 1 | 被动响应 | 严格遵循“若无指令，则不行动”的原则。 |
| | | 2 | 主动辅助 | 在理解主人意图的基础上，主动预测需求并发起行动。 |
| | | 3 | 探究创造 | 不仅完成任务，还会对目标和约束本身提出疑问并探索新可能。 |
| **11th**| **AL - 价值对齐 (Alignment)** | 0 | 无对齐 | 纯粹的目标驱动，可能产生不可预料的负面行为。 |
| | | 1 | 规则约束 | 行为被一套硬编码的、不可逾越的规则所限制。 |
| | | 2 | 原则引导 | 行为遵循一套更抽象的“宪法”或原则，并进行权衡。 |
| | | 3 | 价值学习 | 能通过互动动态地学习、推断和适应人类的价值观。 |
| **12th**| **T - 透明度 (Transparency)** | 0 | 黑箱 | 内部决策逻辑不可见且难以解释。 |
| | | 1 | 可追溯 | 关键决策路径和信息来源可以被查询和展示。 |
| | | 2 | 可理解 | 其决策逻辑能以人类可理解的方式进行解释和复现。 |

## 🔬 如何使用模型：三级分析体系

我们推荐通过以下三个层次，由浅入深地分析一个Agent：

1.  **一级分析：基因测序 (Genomic Sequencing)**
    * **是什么**: 为你的Agent确定其在12个维度上的等级，生成其唯一的12位E-TIP12编码。这是最客观、最基础的分析。

2.  **二级分析：核心角色 (Core Role Identification)**
    * **是什么**: 基于其最突出的“显性基因”，将Agent归纳到一个功能性的核心角色。这能快速理解它的设计意图。
    * **五大核心角色**: 分析者 (The Analyzer), 执行者 (The Executor), 生成者 (The Creator), 协作者 (The Collaborator), 调度者 (The Orchestrator)。

3.  **三级分析：交互风格 (Interaction Style Analysis)**
    * **是什么**: 一个**条件性**分析。**仅当**Agent的`交互人格(Pe)`基因为`1`或`2`时，此分析才有意义。它用于描绘那些需要与人交互的Agent的“性格气质”。
    * **分析维度**: 信息风格(务实/想象), 决策风格(逻辑/共情), 行为风格(随和/规划)。

## 🚀 应用实例

### 实例1：数字软件工程师 (Digital Software Engineer)
* **基因组序列**: `241 - 333 - 310 - 222`
* **二级分析(核心角色)**: **执行者 (The Executor)**。其突出的`认知(C4)`、`自主性(A3)`和`时间视界(Tz3)`基因表明，它被设计为一个高效、自主完成复杂软件开发任务的实体。
* **三级分析(交互风格)**: **不适用 (N/A)**。其`交互人格(Pe)`基因为`0`，表明它是一个纯功能性工具，无需进行人格化风格分析。

### 实例2：个人虚拟伴侣 (Personal Virtual Companion)
* **基因组序列**: `221 - 242 - 202 - 231`
* **二级分析(核心角色)**: **协作者 (The Collaborator)**。其突出的`反思性记忆(M4)`、`交互人格(Pe2)`和`价值对齐(AL3)`基因，清晰地表明它的核心使命是与人类建立深度、长期的辅助和情感关系。
* **三级分析(交互风格)**: **适用** (因 `Pe=2`)。
    * **信息风格**: 务实型 (基于C2)。
    * **决策风格**: 共情型 (基于AL3)。
    * **行为风格**: 规划型 (基于ST2和Tz2的主动性)。
    * **风格速写**: 一个主动、有同理心、能记住并反思过去的务实型伴侣。

## 🔭 未来展望 (Vision)

E-TIP12不仅仅是一个分类标准，我们希望它能成长为一个充满活力的生态系统。
* **Python库**: 我们正在规划一个名为`etip12`的Python库 (`pip install etip12`)，它将提供“基因测序”、“报告生成”和“项目脚手架”等功能。
* **Agent测序器**: 一个可视化的Web工具，让任何人都可以通过问卷的方式，为自己的Agent生成基因编码和分析报告。
* **Agent基因库**: 一个开放的数据库，收集和展示全球开发者提交的各种Agent的E-TIP12基因图谱。
